About AUI

History

In the early days of Atlassian, devs used whatever was handy to slap a UI together. AUI began in 2008 as an answer to the challenges of building multiple products rapidly and efficiently.

Growing from fedex and 20% projects, AUI includes contributions from all Atlassian teams. A dedicated AUI Team was formed in July 2011.

AUI is production-ready and powers the UI of all Atlassian products.

Browser Support

AUI works in IE8, IE9 and the latest versions of Chrome, Firefox, Safari and Opera.

What's Included

AUI comes complete with CSS, Javascript and example templates.

You can test drive these components with the AUI Sandbox and get proformas from the AUI Demo Pages. When you need the deep details, you can refer to the component documentation.

See the How can you get your hands on all this awesome? section below for more details on getting started with AUI.

Messages and Errors

Now that we're getting up to speed, we'll have some messages and states to let the user know how they're doing...

The savepoints are really far apart in this level.

Thank you Mario!

But the Princess is in another castle.

We'll give you a hint to avoid that problem next time.

But first you have to catch the Help Cat with that stupid Wiimote.

You can simply add these messages in your HTML, you can also make them closeable:

        
<div class="aui-message warning shadowed closeable">
    <p class="title">
        <span class="aui-icon icon-warning"></span>
        <strong>It's dangerous to go alone</strong>
    </p>
    <p>Here, take this.</p>
</div><!-- .aui-message -->

Alternatively you can insert them later with Javascript, they'll be closeable by default:

AJS.messages.success({
     title: "Success!",
     body: "This message was created by JS with default options."
});

Note this example will append to the default div, <div id="aui-message-bar"></div> but you can specify the target element if you need to.

Getting Started

How can you get your hands on all this awesome? Chances are you already have it. Open your product, open a Javascript console and run AJS.version - if you get a version number, you're ready to roll!

Otherwise, you can refer to Adding AUI to your application or run up a local copy of the REFAPP:

  1. Set up the Atlassian Plugin SDK. AUI's an Atlassian plugin.
  2. Check out AUI: svn checkout https://studio.atlassian.com/svn/AJS/trunk AJS
  3. From the root of your checkout: mvn clean install
  4. From auiplugin-tests subdirectory: mvn clean install
  5. Go to http://localhost:9999/ajs/ and try the Bootcamp Exercise to get started.

You might want to have a look at the more detailed demos for each of these components, try the Sandbox or view the documentation.
