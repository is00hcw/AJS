Toolbar

3.2.x contains a DRAFT implementation

There are known issues with toolbar dropdowns in IE; and INPUT/BUTTON support has not been completed.

Demonstrates a toolbar with the various button types and states.

Notes

The structure of the toolbar is: aui-toolbar (container) → toolbar-split (left/right or rows) → toolbar-group (contains one set of triggers/buttons) → toolbar-item (contains one single trigger) → toolbar-trigger.

Note the background and border surrounding toolbars is not included in the toolbar component, it's controlled contextuall in the product. The toolbar provides the buttons and button grouping setup.

Button types:

  • Standard/default
  • Primary - a button with an elevated importance, eg. the "publish" button in an editor. Note: there must only be one primary button per toolbar.
  • Dropdown - a button with a dropdown icon, that opens a dropdown menu (one action)
  • Split button - a button with a dropdown icon, where the button performs an action but the dropdown icon opens a menu (two different actions). eg. the "apply colour" button in an editor, which also has a dropdown colour selector.

Button states:

  • Default
  • Active/on - eg. includes buttons which toggle a state, such as the "format: bold" button in an editor
  • Disabled - for buttons which are present/visible but not currently usable. Notes:
    • the disabled class should go on the toolbar-item
    • the disabled class is purely for style and does not disable events- to disable Dropdowns, see instructions for the Dropdown component
  • Hover/focus

All buttons may include text, an icon or both (some buttons may include a second icon to indicate a dropdown). For icon-only buttons the text is hidden in the icon element, so it's still available to screen readers.

The toolbar has been designed to wrap to new lines and increase in height in narrow viewports. Wherever possible, toolbar groups will wrap to new lines as a group (ie. not one button at a time). Left/right split toobars will wrap to two lines when the left/right groups meet; then groups within the left/right splits will wrap to further lines if required.
